================================================================================
              SLACK GATEWAY MODULE — ARCHITECTURAL PLAN
================================================================================

Module Responsibility:
  Listen to Slack events (app mentions, messages), parse user text, route
  requests to the LLM Brain module via a defined interface, and deliver
  text/image responses back to the originating Slack thread.

  This module does NOT contain any LLM logic. It is a thin, resilient
  gateway between Slack and the Brain.

Key separation principle:
  - interfaces/  defines WHAT (abstract contracts, no implementation)
  - core/        defines HOW (concrete logic, depends only on interfaces)
  - models/      defines the DATA that crosses boundaries
  - middleware/   cross-cutting concerns (rate limiting, dedup, auth)

from __future__ import annotations
from dataclasses import dataclass, field
from typing import Optional
from datetime import datetime


@dataclass(frozen=True)
class SlackRequest:
    """Normalized representation of an incoming Slack event.

    Attributes:
        event_id:      Unique Slack event ID (used for dedup).
        team_id:       Workspace ID the event originated from.
        channel_id:    Channel where the event was posted.
        thread_ts:     Thread timestamp; replies go here. Falls back to
                       message_ts if the message is not in a thread.
        message_ts:    Timestamp of the triggering message.
        user_id:       Slack user ID of the person who triggered the event.
        raw_text:      Full text of the Slack message (includes <@BOT_ID>).
        clean_text:    Text with the bot mention stripped and whitespace
                       trimmed. This is what gets sent to the Brain.
        files:         List of file URLs attached to the message (images,
                       documents, etc.).
        received_at:   UTC timestamp of when the gateway received the event.
    """
    event_id: str
    team_id: str
    channel_id: str
    thread_ts: str
    message_ts: str
    user_id: str
    raw_text: str
    clean_text: str
    files: list[str] = field(default_factory=list)
    received_at: datetime = field(default_factory=datetime.utcnow)

from __future__ import annotations
from dataclasses import dataclass, field
from typing import Optional
from enum import Enum, auto


class ResponseType(Enum):
    TEXT = auto()
    IMAGE = auto()
    BLOCKS = auto()       # Slack Block Kit JSON


@dataclass(frozen=True)
class SlackResponse:
    """Payload to send back to Slack.

    Attributes:
        channel_id:     Target channel.
        thread_ts:      Thread to reply in.
        response_type:  What kind of content we're sending.
        text:           Plain text body (required for TEXT, used as
                        fallback for BLOCKS/IMAGE).
        image_url:      Public URL of the image (for IMAGE type).
        blocks:         Block Kit structure (for BLOCKS type).
        ephemeral:      If True, only the requesting user sees the reply.
        user_id:        Required when ephemeral=True.
    """
    channel_id: str
    thread_ts: str
    response_type: ResponseType = ResponseType.TEXT
    text: str = ""
    image_url: Optional[str] = None
    blocks: Optional[list[dict]] = None
    ephemeral: bool = False
    user_id: Optional[str] = None

from __future__ import annotations
from abc import ABC, abstractmethod


class SlackGatewayInterface(ABC):
    """Public contract that the application entrypoint uses to start
    and stop the Slack Gateway.

    Implementations: core/event_listener.py
    """

    @abstractmethod
    async def start(self) -> None:
        """Connect to Slack (Socket Mode or HTTP) and begin listening
        for events. This is a long-running coroutine.

        Raises:
            SlackConnectionError: If the initial connection fails.
        """
        ...

    @abstractmethod
    async def stop(self) -> None:
        """Gracefully disconnect from Slack, draining in-flight
        requests before shutting down.
        """
        ...

    @abstractmethod
    def is_healthy(self) -> bool:
        """Return True if the gateway is connected and processing
        events. Used by health-check endpoints.
        """
        ...

2D. Interface this module EXPECTS from the Brain

from __future__ import annotations
from abc import ABC, abstractmethod
from slack_gateway.models.slack_request import SlackRequest
from slack_gateway.models.slack_response import SlackResponse


class BrainClientInterface(ABC):
    """Contract that the LLM Brain module must satisfy.

    The Slack Gateway calls this interface to delegate user queries.
    The Brain module provides a concrete implementation and injects it
    into the gateway at startup (dependency injection).
    """

    @abstractmethod
    async def process(self, request: SlackRequest) -> SlackResponse:
        """Send the user's cleaned text to the LLM Brain and receive
        a structured response.

        Args:
            request: Parsed and validated SlackRequest.

        Returns:
            SlackResponse ready to be sent back to Slack.

        Raises:
            BrainTimeoutError:   The brain did not respond within the
                                 configured timeout.
            BrainProcessingError: The brain encountered an internal error.
        """
        ...

    @abstractmethod
    async def health_check(self) -> bool:
        """Return True if the Brain is reachable and operational."""
        ...

2E. Exceptions — slack_gateway/errors/exceptions.py
--------------------------------------------------------------------------------

class SlackGatewayError(Exception):
    """Base exception for all Slack Gateway errors."""


class SlackConnectionError(SlackGatewayError):
    """Failed to connect to Slack APIs."""


class EventValidationError(SlackGatewayError):
    """Incoming event failed parsing or signature verification."""


class DuplicateEventError(SlackGatewayError):
    """Event has already been processed (idempotency guard)."""


class BrainTimeoutError(SlackGatewayError):
    """The Brain module did not respond within the allowed window."""


class BrainProcessingError(SlackGatewayError):
    """The Brain module returned an error."""


class RateLimitExceededError(SlackGatewayError):
    """Slack API rate limit hit; retry after backoff."""


class ResponseDeliveryError(SlackGatewayError):
    """Failed to post the response back to Slack."""

3. DATA FLOW & LOGIC
================================================================================

Step-by-step flow for a single incoming app_mention event:

  ┌──────────────────────────────────────────────────────────────┐
  │  SLACK (WebSocket / HTTP)                                    │
  │  Event: app_mention { event_id, channel, text, user, ts }   │
  └──────────────┬───────────────────────────────────────────────┘
                 │
                 ▼
  ┌──────────────────────────────────────────────────────────────┐
  │  1. EVENT LISTENER  (core/event_listener.py)                 │
  │     - Registers Bolt handlers for `app_mention`, `message`   │
  │     - Receives raw event payload from Slack SDK              │
  │     - Immediately ACKs the event (Slack requires <3s ACK)    │
  │     - Hands off to Router asynchronously                     │
  └──────────────┬───────────────────────────────────────────────┘
                 │
                 ▼
  ┌──────────────────────────────────────────────────────────────┐
  │  2. DEDUP MIDDLEWARE  (middleware/dedup.py)                   │
  │     - Check event_id against a TTL cache (e.g., 5 min)       │
  │     - If seen → raise DuplicateEventError → silently drop    │
  │     - If new  → record event_id, proceed                     │
  └──────────────┬───────────────────────────────────────────────┘
                 │
                 ▼
  ┌──────────────────────────────────────────────────────────────┐
  │  3. REQUEST PARSER  (core/request_parser.py)                 │
  │     - Extract fields from raw Slack payload                  │
  │     - Strip bot mention from text: "<@U123> help" → "help"   │
  │     - Resolve thread_ts (use message ts if not in a thread)  │
  │     - Collect any attached file URLs                         │
  │     - Build and return a SlackRequest dataclass              │
  │     - Raise EventValidationError if required fields missing  │
  └──────────────┬───────────────────────────────────────────────┘
                 │
                 ▼
  ┌──────────────────────────────────────────────────────────────┐
  │  4. ROUTER  (core/router.py)                                 │
  │     - Receives validated SlackRequest                        │
  │     - Posts a "thinking" reaction (emoji) to acknowledge      │
  │     - Calls brain_client.process(request) with timeout       │
  │     - On success: receives SlackResponse                     │
  │     - On error:   builds a user-friendly error SlackResponse │
  │     - Removes "thinking" reaction                            │
  │     - Passes SlackResponse to Response Sender                │
  └──────────────┬───────────────────────────────────────────────┘
                 │
                 ▼
  ┌──────────────────────────────────────────────────────────────┐
  │  5. RESPONSE SENDER  (core/response_sender.py)               │
  │     - Reads response_type from SlackResponse                 │
  │     - TEXT:   calls chat_postMessage with text                │
  │     - IMAGE:  calls chat_postMessage with image block         │
  │     - BLOCKS: calls chat_postMessage with blocks payload      │
  │     - Always replies in-thread using thread_ts                │
  │     - If ephemeral: uses chat_postEphemeral instead           │
  │     - Wraps Slack SDK call with rate_limiter middleware        │
  │     - On failure: raises ResponseDeliveryError                │
  └──────────────┬───────────────────────────────────────────────┘
                 │
                 ▼
  ┌──────────────────────────────────────────────────────────────┐
  │  SLACK THREAD                                                │
  │  Bot reply appears in the same thread as the user's message  │
  └──────────────────────────────────────────────────────────────┘


Detailed Router pseudocode (core/router.py):

  async def handle(self, raw_event: dict) -> None:
      # Dedup
      self.dedup.check(raw_event["event_id"])  # raises DuplicateEventError

      # Parse
      request: SlackRequest = self.parser.parse(raw_event)

      # Acknowledge to user
      await self.slack_client.reactions_add(
          channel=request.channel_id,
          timestamp=request.message_ts,
          name="thinking_face"
      )

      try:
          # Call Brain with timeout
          response: SlackResponse = await asyncio.wait_for(
              self.brain_client.process(request),
              timeout=self.config.brain_timeout_seconds   # e.g., 30s
          )
      except asyncio.TimeoutError:
          response = self._error_response(
              request, "Sorry, I'm taking too long to think. Please try again."
          )
      except BrainProcessingError as exc:
          response = self._error_response(
              request, f"Something went wrong on my end. Error: {exc}"
          )
      finally:
          # Remove thinking indicator regardless of outcome
          await self.slack_client.reactions_remove(
              channel=request.channel_id,
              timestamp=request.message_ts,
              name="thinking_face"
          )

      # Send response to Slack
      await self.sender.send(response)


================================================================================
4. ERROR HANDLING
================================================================================

4A. BRAIN TIMEOUT
    ─────────────
    Scenario: brain_client.process() does not return within the configured
              timeout (default: 30 seconds).

    Handling:
      - asyncio.wait_for wraps the brain call with a hard timeout.
      - On TimeoutError, the router builds a friendly error SlackResponse:
            "I'm taking too long to respond. Please try again shortly."
      - The "thinking" reaction is removed so the user isn't left hanging.
      - The timeout event is logged at WARNING level with the event_id
        and elapsed time for observability.
      - No retry is attempted — the user can re-trigger manually.


4B. SLACK API RATE LIMITS (HTTP 429)
    ─────────────────────────────────
    Scenario: Slack returns 429 Too Many Requests with a Retry-After header.

    Handling (middleware/rate_limiter.py):
      - Wraps every outbound Slack API call.
      - On 429: reads the Retry-After header (seconds).
      - Applies exponential backoff with jitter, capped at 3 retries.
      - Retry sequence example: wait Retry-After + jitter, then retry.
      - If all retries exhausted: raise RateLimitExceededError.
      - The router catches this and logs at ERROR level. The user's
        message is NOT silently dropped — a fallback error is logged
        and an alert is emitted if alerting is configured.
      - Per-method rate tracking: Slack rate-limits are per-method
        (e.g., chat.postMessage has a different limit than reactions.add).
        The rate limiter tracks limits per method independently.


4C. DUPLICATE EVENTS (SLACK RETRIES)
    ──────────────────────────────────
    Scenario: Slack retries delivery of an event the gateway already
              processed (e.g., due to slow ACK).

    Handling (middleware/dedup.py):
      - Maintains a TTL cache (in-memory dict or Redis for multi-instance)
        keyed by event_id with a 5-minute TTL.
      - If the event_id is already in the cache → raise DuplicateEventError.
      - The event listener catches DuplicateEventError silently (no user
        message, no error log — just a DEBUG log).


4D. MALFORMED / INVALID EVENTS
    ───────────────────────────
    Scenario: Incoming event is missing required fields, has unexpected
              structure, or fails signature verification.

    Handling:
      - request_parser.py validates required fields (channel, user, text).
      - Missing fields → raise EventValidationError with details.
      - signature_verifier.py (HTTP mode) checks X-Slack-Signature against
        the signing secret. Invalid signature → reject with 401, log at
        WARNING.
      - These errors are never surfaced to the Slack user (they indicate
        infrastructure issues, not user mistakes).


4E. SLACK CONNECTION LOSS
    ─────────────────────
    Scenario: WebSocket connection drops (Socket Mode) or Slack becomes
              unreachable.

    Handling:
      - The Slack Bolt SDK handles reconnection automatically in Socket
        Mode with built-in exponential backoff.
      - The gateway's is_healthy() method returns False during
        disconnection, allowing external health checks to detect the
        outage.
      - Events received during reconnection are not lost — Slack
        redelivers unacknowledged events.


4F. RESPONSE DELIVERY FAILURE
    ──────────────────────────
    Scenario: chat_postMessage fails for reasons other than rate limiting
              (e.g., channel archived, bot removed from channel, network
              error).

    Handling:
      - response_sender.py catches Slack API errors.
      - Transient errors (network): retry up to 2 times with backoff.
      - Permanent errors (channel_not_found, not_in_channel): log at
        ERROR with full context, do NOT retry.
      - Raise ResponseDeliveryError so the router can log the failure.

Expected configuration (loaded from environment variables):

    SLACK_BOT_TOKEN          Bot OAuth token (xoxb-...)
    SLACK_APP_TOKEN          App-level token for Socket Mode (xapp-...)
    SLACK_SIGNING_SECRET     For HTTP-mode signature verification
    BRAIN_TIMEOUT_SECONDS    Max wait for brain response (default: 30)
    RATE_LIMIT_MAX_RETRIES   Max retries on 429 (default: 3)
    DEDUP_TTL_SECONDS        Event dedup cache TTL (default: 300)
    LOG_LEVEL                Logging verbosity (default: INFO)

These are loaded via a Pydantic BaseSettings class for validation
and type coercion.

6. DEPENDENCY INJECTION PATTERN
================================================================================

The gateway is assembled at application startup:

    from slack_gateway.core.event_listener import SlackEventListener
    from slack_gateway.core.router import Router
    from slack_gateway.core.request_parser import RequestParser
    from slack_gateway.core.response_sender import ResponseSender
    from slack_gateway.middleware.rate_limiter import RateLimiter
    from slack_gateway.middleware.dedup import DeduplicationGuard
    from slack_gateway.config import GatewayConfig
    # Brain implementation comes from OUTSIDE this module
    from brain_module import ConcreteBrainClient

    config = GatewayConfig()                           # from env vars
    brain = ConcreteBrainClient(...)                   # injected dependency
    rate_limiter = RateLimiter(config)
    dedup = DeduplicationGuard(config)
    parser = RequestParser(bot_user_id="U_BOT_ID")
    sender = ResponseSender(slack_client, rate_limiter)
    router = Router(parser, dedup, brain, sender, config)
    gateway = SlackEventListener(slack_app, router)

    await gateway.start()

This ensures:
  - The gateway never imports concrete Brain code.
  - The Brain module only needs to implement BrainClientInterface.
  - Testing is trivial: inject a mock BrainClientInterface.
