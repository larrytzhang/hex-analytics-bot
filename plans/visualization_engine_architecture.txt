================================================================================
            VISUALIZATION ENGINE MODULE — ARCHITECTURAL PLAN
================================================================================

Module Scope
------------
Take structured data (JSON/dicts) and metadata (title, axis labels, chart type),
generate a publication-quality chart, and return the result as a file path or
byte buffer. No awareness of Slack, LLMs, or databases.


================================================================================
1. FOLDER STRUCTURE
================================================================================

visualization_engine/
├── __init__.py                  # Public API re-exports
├── interface.py                 # Abstract base classes and public type defs
├── engine.py                    # Concrete ChartEngine implementation
├── chart_types/
│   ├── __init__.py
│   ├── bar.py                   # Bar / grouped-bar / stacked-bar renderer
│   ├── line.py                  # Line / multi-line renderer
│   ├── pie.py                   # Pie / donut renderer
│   ├── scatter.py               # Scatter / bubble renderer
│   └── table.py                 # Styled table-as-image renderer
├── inference.py                 # Auto-detect best chart type from data shape
├── styling.py                   # Theme, color palettes, font config
├── export.py                    # Save to disk or return bytes buffer
├── validators.py                # Input validation and sanitisation
├── errors.py                    # Custom exception hierarchy
└── config.py                    # Module-level defaults (DPI, size, theme)

Separation of concerns:
  - interface.py   = WHAT the module promises (contracts only, no logic).
  - engine.py      = HOW it fulfils those promises (orchestration).
  - chart_types/*  = Individual rendering strategies, one per file.
  - inference.py   = Decision logic kept isolated for testability.
  - export.py      = Output serialisation, decoupled from rendering.

---------- errors.py ----------

class VisualizationError(Exception):
    """Base exception for the visualization engine."""

class EmptyDataError(VisualizationError):
    """Raised when the input dataset contains no plottable rows."""

class DataTypeMismatchError(VisualizationError):
    """Raised when column types are incompatible with the requested chart."""

class UnsupportedChartTypeError(VisualizationError):
    """Raised when an unknown chart type string is provided."""

---------- interface.py ----------

from abc import ABC, abstractmethod
from dataclasses import dataclass
from enum import Enum, auto
from io import BytesIO
from pathlib import Path
from typing import Any, Dict, List, Optional, Union

class ChartType(Enum):
    BAR = auto()
    GROUPED_BAR = auto()
    STACKED_BAR = auto()
    LINE = auto()
    MULTI_LINE = auto()
    PIE = auto()
    DONUT = auto()
    SCATTER = auto()
    TABLE = auto()
    AUTO = auto()               # let the engine decide

@dataclass
class ChartRequest:
    """
    Everything the caller must provide to produce a chart.

    Attributes:
        data:         List of dicts (row-oriented) or dict of lists
                      (column-oriented). Each key = column name.
        chart_type:   Explicit type or AUTO for inference.
        title:        Chart title string.
        x_label:      Label for the x-axis (ignored for pie/table).
        y_label:      Label for the y-axis (ignored for pie/table).
        x_column:     Key in data to use as x-axis.
        y_columns:    Key(s) in data to use as y-axis / values.
        group_column: Optional grouping key (for grouped/stacked bars,
                      multi-line).
        config:       Rendering preferences; None = module defaults.
    """
    data: Union[List[Dict[str, Any]], Dict[str, List[Any]]]
    chart_type: ChartType = ChartType.AUTO
    title: str = ""
    x_label: str = ""
    y_label: str = ""
    x_column: Optional[str] = None
    y_columns: Optional[List[str]] = None
    group_column: Optional[str] = None
    config: Optional["ChartConfig"] = None

@dataclass
class ChartResult:
    """
    What the engine hands back after rendering.

    Attributes:
        image_bytes:  In-memory PNG/SVG bytes (always populated).
        file_path:    Disk path if the caller asked to save; else None.
        chart_type:   The chart type that was actually rendered
                      (useful when AUTO was requested).
        metadata:     Any extra info (e.g. inferred axis ranges).
    """
    image_bytes: bytes
    file_path: Optional[Path] = None
    chart_type: ChartType = ChartType.BAR
    metadata: Dict[str, Any] = None

class ChartEngineInterface(ABC):
    """Contract every engine implementation must satisfy."""

    @abstractmethod
    def render(self, request: ChartRequest) -> ChartResult:
        """
        Generate a chart from structured data.

        Args:
            request: A fully-populated ChartRequest.

        Returns:
            ChartResult with image bytes and optional file path.

        Raises:
            EmptyDataError:             if data has zero plottable rows.
            DataTypeMismatchError:      if column types conflict with chart.
            UnsupportedChartTypeError:  if chart_type is unrecognised.
        """
        ...

    @abstractmethod
    def render_to_file(
        self,
        request: ChartRequest,
        output_path: Union[str, Path],
    ) -> ChartResult:
        """
        Render and persist the chart to disk.

        Args:
            request:     A fully-populated ChartRequest.
            output_path: Destination file path.

        Returns:
            ChartResult with file_path set.
        """
        ...

    @abstractmethod
    def supported_chart_types(self) -> List[ChartType]:
        """Return the chart types this engine can produce."""
        ...

================================================================================
3. DATA FLOW & LOGIC
================================================================================

Step-by-step pipeline executed inside ChartEngine.render():

  ┌──────────────┐
  │ ChartRequest  │
  └──────┬───────┘
         │
         ▼
  ┌──────────────────────────────────────┐
  │ 1. VALIDATE INPUT                    │
  │    - Normalise data to column-        │
  │      oriented dict (if row-oriented). │
  │    - Reject empty datasets            │
  │      → raise EmptyDataError.          │
  │    - Verify x_column and y_columns    │
  │      exist in the data keys.          │
  │    - Check column dtypes are          │
  │      compatible with chart_type       │
  │      → raise DataTypeMismatchError.   │
  └──────┬───────────────────────────────┘
         │
         ▼
  ┌──────────────────────────────────────┐
  │ 2. INFER CHART TYPE (if AUTO)        │
  │    (inference.py)                     │
  │                                       │
  │    Heuristics:                        │
  │    a) 1 categorical col + 1 numeric   │
  │       col → BAR                       │
  │    b) 1 datetime/ordered col +        │
  │       1+ numeric cols → LINE /        │
  │       MULTI_LINE                      │
  │    c) 2 numeric cols → SCATTER        │
  │    d) 1 categorical + 1 numeric       │
  │       + values sum ≈ 100% → PIE       │
  │    e) All categorical → TABLE         │
  │    f) Fallback → BAR                  │
  └──────┬───────────────────────────────┘
         │
         ▼
  ┌──────────────────────────────────────┐
  │ 3. DISPATCH TO RENDERER              │
  │    - Registry maps ChartType →       │
  │      renderer callable in            │
  │      chart_types/*.                   │
  │    - Renderer receives:              │
  │        (data_dict, request, config)  │
  │    - Renderer returns a              │
  │      matplotlib.figure.Figure.       │
  └──────┬───────────────────────────────┘
         │
         ▼
  ┌──────────────────────────────────────┐
  │ 4. APPLY STYLING                     │
  │    (styling.py)                       │
  │                                       │
  │    - Load theme (colors, fonts,       │
  │      gridlines, spine visibility).    │
  │    - Set title, axis labels, legend.  │
  │    - Apply tight_layout / constrained │
  │      layout for clean spacing.        │
  │    - Remove chart junk (Tufte style): │
  │      mute gridlines, soften spines,   │
  │      use muted palette.               │
  └──────┬───────────────────────────────┘
         │
         ▼
  ┌──────────────────────────────────────┐
  │ 5. EXPORT                            │
  │    (export.py)                        │
  │                                       │
  │    - Render figure → BytesIO buffer   │
  │      at configured DPI and format.    │
  │    - If output_path requested, also   │
  │      write buffer to disk.            │
  │    - Close figure to free memory.     │
  │    - Return ChartResult.              │
  └──────────────────────────────────────┘

    ) -> matplotlib.figure.Figure:
        ...


================================================================================
4. ERROR HANDLING STRATEGY
================================================================================

Principle: Never crash. Return a clear, actionable error that the calling
module can relay or log.

4.1 Empty Datasets
    ─────────────
    Detected in: validators.py — check len(data) == 0 or all columns empty.
    Action:      Raise EmptyDataError("No plottable data rows found.
                 Received <N> columns, all with 0 non-null values.")

4.2 Mismatched Data Types
    ─────────────────────
    Detected in: validators.py — after normalisation, sample each column's
                 dtype. Compare against chart type requirements:
                   - BAR x-axis: categorical or string
                   - LINE x-axis: numeric, datetime, or ordinal
                   - SCATTER: both axes numeric
                   - PIE values: numeric
    Action:      Raise DataTypeMismatchError with a message naming the
                 offending column, its detected type, and what the chart
                 type requires.
                 Example: "Column 'revenue' has dtype 'str', but SCATTER
                 requires numeric y-axis values."

4.3 Unknown Chart Type
    ──────────────────
    Detected in: engine.py — lookup in _RENDERERS.
    Action:      Raise UnsupportedChartTypeError listing valid options.

4.4 Partial / Malformed Data
    ────────────────────────
    - Rows with None values: drop silently and add a warning to
      ChartResult.metadata["warnings"].
    - Columns named in x_column / y_columns that do not exist: raise
      KeyError wrapped in DataTypeMismatchError.
    - Mixed-type columns (e.g. ["1", 2, "three"]): attempt numeric
      coercion; if >50% fail, treat as categorical.

4.5 Rendering Failures
    ──────────────────
    - Matplotlib errors are caught in engine.py, wrapped in
      VisualizationError with the original traceback attached.
    - Figure is always closed in a finally block to prevent memory leaks.

4.6 File I/O Failures
    ─────────────────
    - export.py catches OSError / PermissionError when writing to disk,
      wraps in VisualizationError("Failed to write chart to <path>: ...").
    - The in-memory buffer is still returned so the caller can retry or
      use the bytes directly.


================================================================================
5. ADDITIONAL DESIGN NOTES
================================================================================

5.1 Thread Safety
    Matplotlib is not thread-safe when using the default backend. The engine
    will use the "Agg" (non-interactive) backend and create a new Figure per
    call — no shared global state.

5.2 Testing Strategy
    - Unit tests per renderer with small fixture datasets.
    - Property tests on inference.py (random data shapes → valid ChartType).
    - Integration test: ChartRequest → ChartResult with image byte
      verification (non-zero length, valid PNG header).
    - Snapshot tests for styling regressions (compare against golden images
      with a pixel-diff tolerance).

5.3 Extensibility
    Adding a new chart type:
      1. Create a new file in chart_types/.
      2. Implement the standard renderer signature.
      3. Add the ChartType enum member.
      4. Register in _RENDERERS.
      5. Add inference rules in inference.py (optional).

5.4 Dependencies
    Required:  matplotlib >= 3.7
    Optional:  seaborn (for statistical chart types, imported lazily)
               pandas (accepted as input, converted internally)
